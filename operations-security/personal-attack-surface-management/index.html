Personal Attack Surface Management

This section will cover data removal/erasure in various categories such as old drives, data brokers, breach compromise notification services, and online accounts. This will allow you to decrease your personal digital attack surface.

Drives​

People tend to forget to clean their drives before selling or donating their used drives. This allows sensitive and personal data to still be on the drive, allowing the next owner of the drive to use that data as they please. Even if you were to "delete" the files on your system, they would end up in a folder called ".Trash-1000" (where 1000 is the UID). From there the file(s) can be recovered easily. To fully remove those files, you would need to run sudo rm -rf .Trash-1000. Hard Drives (HDs) and Solid State Drives (SSDs) have different mechanisms, so the process of erasing/wiping those systems are different as well.

Hard Drive​

For Hard Drives, the best solution I was able to find was GitHub - PartialVolume/shredos.x86_64. This works great with Ventoy (alternate link: GitHub - ventoy/Ventoy: A new bootable USB solution.) so you don't have to overwrite your whole drive and keep only one OS on it. Another alternative that works well is DBAN (DBAN has not been updated since 2023-07-02). ShredOS uses Nwipe on the back-end, and supports the following erasure methods:

  • Fill With Zeros - Fills the device with zeros (0x00), one round only.
  • Fill With Ones - Fills the device with ones (0xFF), one round only.
  • RCMP TSSIT OPS-II - Royal Canadian Mounted Police Technical Security Standard, OPS-II
  • DoD Short - The American Department of Defense 5220.22-M short 3 pass wipe (passes 1, 2 & 7).
  • DoD 5220.22M - The American Department of Defense 5220.22-M full 7 pass wipe.
  • Gutmann Wipe - Peter Gutmann's method (Secure Deletion of Data from Magnetic and Solid-State Memory).
  • PRNG Stream - Fills the device with a stream from the PRNG.
  • Verify Zeros - This method only reads the device and checks that it is filled with zeros (0x00).
  • Verify Ones - This method only reads the device and checks that it is filled with ones (0xFF).
  • HMG IS5 enhanced - Secure Sanitisation of Protectively Marked Information or Sensitive Information

I have had success with the Gutmann Wipe method, which does 35 passes [2]. Feel free to use an erasure method based on your need and/or threat model. Other command-line methods can be found at Securely wipe disk - ArchWiki.

Solid State Drives​

Solid State Drives have a more different approach. Some Motherboards UEFI/BIOS systems have a "secure erase" feature that allow you to wipe an SSD straight from the UEFI. However, most OEM UEFI/BIOS systems are proprietary and have closed-source code. As such, open-source alternatives are highly encouraged by the privacy community. One of these alternatives is GitHub - linux-nvme/nvme-cli: NVMe management command line interface. This can easily be installed on a Debian-based system with sudo apt install nvme-cli. The following commands would erase your SSD [3]:

nvme sanitize -a 2 /dev/nvme0n1
# OR
nvme format /dev/nvme0 -s 2 -n 1

To check the sanitation status, run nvme sanitize-log /dev/nvme0. The following is the output expected when the program is running compared to when it has been completed [9]:

Running:

Sanitize Progress                      (SPROG) :  16704
Sanitize Status                        (SSTAT) :  0x2
...

Complete:

Sanitize Progress                      (SPROG) :  65535
Sanitize Status                        (SSTAT) :  0x101
...

After the sanitation is completed, you can verify the end result by running dd if=/dev/nvme0n1 bs=8192 status=progress | hexdump.

Data Erasure Workflow​

The following are steps summarized and modified from the Data Erasure - Divested Computing page (It turned out to be a great resource for this section):

  • Identify your drive (/dev/sdx OR /dev/nvme0n1) by using GNOME Disks, KDE Partition Manager, GParted, or lsblk.

  • Discard All Data

    • Check if drive allows for discard: lsblk /dev/nvme0n1 --discard. If output of DISC-GRAN is 0, this will not work for you.

      • If it does run blkdiscard --force --verbose --secure /dev/nvme0n1.

      • If that does not work, run blkdiscard --force --verbose /dev/nvme0n1.

  • Perform a traditional erase

    • shred --iterations 1 --verbose /dev/nvme0n1

  • Erase All Data

    • NVMe Drives (for following commands leave the NVMe namespace off (ex. /dev/nvme0n1 is now /dev/nvme0)

      • Block Erase: nvme sanitize --sanact 0x02 /dev/nvme0.

      • If that does not work, then run nvme format --namespace-id 1 --ses 1 /dev/nvme0.

      • Verify the sanitation of the drive: dd if=/dev/nvme0n1 bs=8192 status=progress | hexdump.

    • Other Drives (Avoid USB Enclosures for this)

      • Check if hdparm commands we need are supported: hdparm -I /dev/sdX | grep frozen. If the output is nothing, this step will not be completed. I would recommend using ShredOS instead here.

        • If output is "frozen", put your computer to sleep, wake it up, and then run the command again.

        • If output is "not frozen", then continue.

      • Set a temporary password hdparm --user-master u --security-set-pass password /dev/sdX.

      • Initiate the erase: hdparm --user-master u --security-erase-enhanced password /dev/sdX.

    • If neither of these steps worked for you, on the blog they mention to run blkdiscard again. I would recommend doing that or using ShredOS and leveraging the Guttman wipe as it mentions it can be used for Solid State memory.

Just another reminder, the previous workflow heavily relied on information from Data Erasure - Divested Computing.

Data Brokers​

Data Brokers have been a threat to privacy as these companies not only sell your data to third-parties, but also make the data available for anyone who has access to the internet to access your data. If an individual has malicious intent towards you they can use this data to find you. If a site doesn't have the information on you, another site will. As such it is imperative to remove data from these sites as soon as possible. There are multiple ways to go about this, but I will walk through the steps that have assisted me in this removal the best.

Search Engines​

Search engines are someones first step in looking for you. It is similar to how you would look for a website for a company: If you just know the name of a company "Codeberg", searching that on any search engine will show you that the Codeberg organization is using https://codeberg.org/ as their website. We can use this same approach for individuals. By searching "Your [Full Name] Address", you can see all of the people search sites (part of the data broker ecosystem) that are more than happy to provide anybody your address. [The Paranoid’s Bible 2.0](The Paranoid’s Bible 2.0 - The Paranoid's Bible: An anti-dox effort.) has some great examples of what to use to search for information on yourself [6]:

  1. Full name (First, Middle, Last and/or marks JR/SR/2nd/ 3rd…etc)
  2. First + last name
  3. First name
  4. Last name/home address
  5. Name combinations + birth-date
  6. Name combinations + home/street address
  7. Name combinations + school
  8. Name combinations + work
  9. Name combinations + vote, voted or voting
  10. Street address
  11. Relatives names + name combinations
  12. Usernames + name combinations
  13. Usernames
  14. E-mail addresses
  15. E-mail addresses + usernames
  16. Websites or accounts + usernames
  17. Websites or accounts + e-mail addresses
  18. Usernames + city or state
  19. Name combinations + city or state
  20. The above + IP addresses or ISP
  21. The above + known leak or dump searches (E.G: Have I Been PWNED)

Due to this, the first item we need to do is to use Google to remove all sites indexed that include our information. Google offers https://myactivity.google.com/results-about-you, which allows you to remove your personal data from Google by inputting your own information in. Only give Google data that is already public (ex. on people search sites already). I would recommend using your personal gmail account, if you have one already.

Side-note: I recommend not deleting your Gmail, even if you are not using it. This allows you to use this same email later on - when sending emails to request data brokers to remove your data from their website.

Bing has something similar to Google: Bing Webmaster Tools. These will most likely be people's first attempt to try to find you. In the next step, we will request removals from the data brokers directly. If your data is still indexed on search engines after the data has been removed from the data brokers site, request the search engine to index that site again (or just tell it the site is no longer functional).

Data Broker Websites​

There are a couple of great "mega-lists" of data broker websites. You simply go to each of the data broker websites, and then look for your own information. If you are able to find something on yourself, you ask them to remove it. If there is nothing, then no work is needed there. Some data brokers use online forms for opt-outs, others require an email. The most popular of these "mega-lists" is [Michael Bazzell's Workbook](IntelTechniques Data Removal Workbook). Another one I have seen is from Yael: GitHub - yaelwrites/Big-Ass-Data-Broker-Opt-Out-List. PrivacyGuides has mentioned their list on their site: Data Removal Services - Privacy Guides. My current recommendation is to just use Michael's Workbook, and then use the search query examples mentioned in the "Search Engines" section above to look for information on you. There are some sites that are not in any of the "mega-lists", but are still data broker sites. You will have to manually find the opt-out links for those.

Some data broker websites will require an ID for you to prove that you are who you say you are. For this, I take a picture of my ID, open it in GIMP (locally), then put black boxes on all of the information not available on the website. This includes your picture, your ID number, ID Expiration date, etc. Do not give them any more information than they already have. I take this one step further, and take a screenshot of the image, so there is only one "layer" in the image. If they require a head-shot, I would recommend using an image from https://www.thispersondoesnotexist.com/ as they most likely do not know what you look like (CAVEAT: This could also make things worse for you if the organization ends up figuring out it is not actually you).

Breach Compromise Notification Services​

There are websites that tell you what breaches your data is part of, and some even go farther to tell you part of your password in the breach. The downside to this, is that everybody else can see that information about us as well, as long as they know your email address. As such, removing our data from these sites is important as well. The following is copied verbatim from Michael Bazzell's book "Extreme Privacy 5th Edition" pages 433-434:

Have I Been Pwned:
Conduct a search of your email address or username at https://haveibeenpwned.com/OptOut.
Click the link in the email confirmation.
Choose the "Remove email address completely" option.

Dehashed:
Create a free burner account at https://dehashed.com.
Conduct a search of your email address or username.
Click "Request entry removal" link below each result.
Click the confirmation link within each email.

Leakcheck:
Register for a free burner account at https://leakcheck.net.
Conduct a search of your email address.
If entries are present , send an email from the exposed address with a written removal request to
removal@leakcheck.net.
Click the link within the confirmation email.

PSBDMP:
Navigate to the IntelTechniques Breach Tool at https://inteltechniques.com/tools/Breaches.html.
Search your email address at the first "PSBDMP" option.
If any results are present, send an email to admin@psbdmp.ws requesting removal.

IntelX:
Conduct a search of your email address at https://intelx.io.
Navigate to https://intelx.io/abuse.
Paste any URLs associated with the found content "Full Data" links.
Submit request.

HudsonRock:
Navigate to the IntelTechniques Breach Tool at https://inteltechniques.com/tools/Breaches.html.
Search your email address at the first "HudsonRock" option.
If any results are present, send an email to hello@hudsonrock.com requesting removal. If that fails, use the
contact page at https://www.hudsonrock.com/contact.

Leak-Lookup:
Register for a free burner account at https://leak-lookup.com.
Conduct a search of your email address.
If entries are present , send an email from the exposed address with a written removal request to info@leak-
lookup.com.

HackCheck:
Register for a free burner account at https://hackcheck.io.
Conduct a search of your email address.
If entries are present , send an email from the exposed address with a written removal request to
support@hackcheck.io.

Breach Directory:
Navigate to https://breachdirectory.org.
Search your email address.
If any results are present, begin the removal process at https://breachdirectory.org/deletemydata.

LeakPeek:
Navigate to https://leakpeek.com.
Search your email address.
If any results are present, send an email to support@LeakPeek.com requesting removal.

Online Accounts​

In the modern world, there are accounts required for everything. From the thermostat, to your car, to your fridge, etc. As such, it is important to remove accounts when they are no longer needed. The question then arises, how do we know what accounts we have? There are multiple ways to do this. One easy way is to look in your password manager to see all of the accounts you have. Another way to do this is to use an OSINT tool such as GitHub - WebBreacher/WhatsMyName. Alternatives to this would include GitHub - sherlock-project/sherlock and GitHub - soxoj/maigret. You can use these tools to hunt for usernames you might have had in the past to see what accounts of yours are still in the open. From there all you have to do is to change the information on the website, and then delete the account. There is a possibility that while deleting an account, an organization might archive it - although this might be getting into paranoid territory. Regardless, you can delete the account from the sites you no longer need this information from.
